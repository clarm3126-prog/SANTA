SANTA (산타)

이 시안은 단순한 동행 앱이 아니라 등산 커뮤니티 + 지도 기반 코스 탐색 + 필요할 때 쓰는 안전 동행 을 한 화면 안에 묶는 방향으로 다시 정리했습니다.

Core 1
오늘 갈 산을 결정하게 해주는 지도 기반 코스 탐색과 입구별 브리핑
Core 2
예쁜 사진보다 먼저 보이는 최근 7일 후기, 질문, 위험 구간 커뮤니케이션
Core 3
동행은 메인이 아니라 산별 커뮤니티가 활성화된 뒤 자연스럽게 붙는 부가 기능
Core 4
실명 인증, 후기 신뢰도, 위치 공유, 귀가 체크를 동행 전환 순간에 강하게 노출
포지셔닝 수정
동행을 첫 화면의 이유로 두지 않고, 산행 전 판단과 산행 후 대화가 가장 잘 되는 커뮤니티 서비스로 표현을 정리했습니다.
정보 구조 수정
홈 진입 순서를 산 탐색, 커뮤니티 브리핑, 지도 체크, 동행 보조 기능으로 재정렬해 실제 획득 흐름에 맞췄습니다.
웹 우선 UX
모바일 앱 셸은 유지하되 데스크톱에서도 전략, 커뮤니티 구조, 산 상세 탐색 흐름을 함께 검토할 수 있도록 양쪽 레이아웃으로 구성했습니다.
왜 이 방향이 맞는가: 사용자는 먼저 산과 코스를 결정하고, 그다음 최근 후기와 질문을 보고, 마지막에 필요하면 동행을 찾습니다.
이 단계의 의미: 다음 React 전환 전에 커뮤니티 중심 IA와 지도형 상세 구조가 실제 화면에서 설득력 있는지 먼저 검증합니다.

👥 동행 신청하기
이 서비스의 기본은 산별 커뮤니티이며, 동행은 필요할 때 쓰는 보조 기능입니다. 신청 전에 최근 후기, 코스 난이도, 위치 공유 여부를 확인하세요.

✅ 실명 인증 완료
✅ 프로필 사진 등록
✅ 자기소개 30자 이상
✅ 산행 경험 1회 이상
✅ 위치 공유 및 귀가 체크 동의
동행 중에는 위치 공유를 켜고, 노쇼 또는 비매너 행동 시 서비스 이용이 제한될 수 있습니다.
🏔 이 산 같이 가실 분?
북한산 커뮤니티에서 동행으로 전환된 글이 현재 3건 있습니다. 기본 추천은 3~5명 그룹이며, 후기와 신뢰도가 높은 호스트를 우선 노출합니다.
