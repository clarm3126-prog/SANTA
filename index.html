SANTA (산타)

이 시안은 단순한 등산 SNS가 아니라 정확한 코스 탐색 + 구조화된 후기 + 안전 기반 동행 매칭 을 한 화면 안에 묶는 방향으로 다시 정리했습니다.

Core 1
램블러형 탐색 정확도에 가까운 코스 탐색과 위험 구간 요약
Core 2
인스타형 피드가 아니라 산행 의사결정에 필요한 구조화된 후기
Core 3
소개팅이 아니라 그룹 중심의 안전 동행 매칭
Core 4
실명 인증, 후기, 위치 공유, 귀가 체크를 기본 UX로 노출
포지셔닝 수정
데이트 느낌을 전면에 두지 않고, 초보도 안심하고 참여할 수 있는 소규모 동행 플랫폼으로 표현을 정리했습니다.
코드 구조 개선
내비게이션, 탭, 피드 카드, 동행 모집글을 하드코딩하지 않고 데이터 기반으로 렌더링하도록 구조를 바꿨습니다.
웹 우선 UX
모바일 앱 셸은 유지하되 데스크톱에서도 전략과 프로토타입을 동시에 검토할 수 있도록 양쪽 레이아웃으로 구성했습니다.
왜 이 방향이 맞는가: 등산은 기록앱만으로 차별화하기 어렵고, 안전과 사람 연결을 동시에 해결해야 전환 이유가 생깁니다.
이 단계의 의미: 다음 React 전환 전에 더미 데이터를 한곳에 모아 상태와 렌더링 구조를 먼저 검증합니다.
